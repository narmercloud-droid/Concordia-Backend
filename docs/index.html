Authentication (BearerAuth)

All protected endpoints use BearerAuth. Provide an HTTP header:

Authorization: Bearer <token>

Admin Authentication Notes

Admin endpoints require an administrator token obtained via the admin login endpoint. Typical flow:

  1. Call the admin login endpoint with credentials to receive a token.
  2. Include the token in the Authorization: Bearer <token> header for subsequent admin calls.

Admin Examples

TypeScript (fetch)

import ConcordiaAdminClient from '../sdks-admin/typescript/dist/client';
  const client = new ConcordiaAdminClient('https://api.concordia.app');
  const auth = await client.adminAuthLogin({ username: 'admin', password: 'secret' });
  const token = auth.token;
  const admin = new ConcordiaAdminClient('https://api.concordia.app', token);
  const orders = await admin.adminListOrders();
  console.log(orders);

Python (requests)

from concordia_admin_client import ConcordiaAdminClient
  client = ConcordiaAdminClient()
  auth = client.adminAuthLogin({'username':'admin','password':'secret'})
  token = auth.get('token')
  admin = ConcordiaAdminClient(token=token)
  orders = admin.adminListOrders()
  print(orders)

Examples

  • TypeScript (fetch): headers: { Authorization: `Bearer ${token}` }
  • Python (requests): headers={'Authorization': f'Bearer {token}'}
  • C# (.NET): client.DefaultRequestHeaders.Authorization = new AuthenticationHeaderValue("Bearer", token);
  • Swift (URLSession): request.setValue("Bearer \(token)", forHTTPHeaderField: "Authorization")
  • Kotlin (Ktor): set the header Authorization: Bearer <token> on requests.

Deprecation Policy

Breaking changes will be announced with a 90-day notice. Deprecated endpoints will be marked in the OpenAPI spec with deprecated: true. SDKs will be updated prior to removal and communicated in the changelog.

Error Handling

All error responses follow a standard shape with `error`, `message`, and `code` fields. Example:

{ "error": "BadRequest", "message": "Invalid payload", "code": "INVALID_PAYLOAD", "details": null }

Rate Limits

Responses include rate-limit headers: X-RateLimit-Limit, X-RateLimit-Remaining, and X-RateLimit-Reset. Clients should back off when X-RateLimit-Remaining is low.

Retry Guidance

For idempotent operations, consider retrying on transient failures (5xx) with exponential backoff. Respect Retry-After when provided (429 responses).

Mock Server & Testing

Run the local mock server for integration and SDK testing on port 4000. See /mock-server/README.md for usage and webhook simulation.

Response Overrides

Use the runtime override API at /__overrides to force specific responses for routes. POST a JSON map of overrides to merge, or DELETE to clear all overrides.
